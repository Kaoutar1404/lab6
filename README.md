# 🍕 Étude de Cas : Application Android "Pizza Gourmet"

## 📑 Présentation du Projet
Ce projet consiste en la réalisation d'une application mobile native développée sous **Android Studio** en langage **Java**. L'objectif est de fournir une interface fluide pour la consultation d'un catalogue de recettes, intégrant une architecture modulaire et une gestion optimisée des listes.


## 🛠️ Architecture Technique
Le projet respecte une organisation rigoureuse découpée en packages fonctionnels pour faciliter la maintenance :

* **`com.example.pizza.model`** : Définition de l'objet métier (Pizza/Produit).
* **`com.example.pizza.dao`** : Interface d'accès aux données (Pattern IDao).
* **`com.example.pizza.service`** : Logique de gestion et stockage en mémoire (Singleton).
* **`com.example.pizza.ui`** : Regroupement des interfaces utilisateurs (Splash, List, Details).

## 🚀 Fonctionnalités Implémentées
1. **Initialisation** : Écran de démarrage (SplashActivity) pour l'identité visuelle.
2. **Affichage Dynamique** : Utilisation d'une `ListView` avec un `Adapter` sur mesure pour un rendu personnalisé.
3. **Navigation Intelligente** : Passage des données entre les activités via les `Intents`.
4. **UI Moderne** : Design épuré basé sur des cartes pour une meilleure lisibilité.


## 📁 Vue d'ensemble des sources
L'arborescence du projet suit les standards de l'industrie mobile :
- `res/layout` : Définition des interfaces XML.
- `res/drawable` : Ressources graphiques et icônes.
- `AndroidManifest.xml` : Configuration des permissions et déclaration des activités.
